This module implements a Texas Instruments MSP430F1611
microcontroller in software. See the documentation for
MSP430Type and new() for details.
 
This module defines an exception named MSP430Error and
an alias named error.
 
_soft430.error is raised when an error condition
occurs in the simulator. Errors generated at the C
level will raise this exception with three args:
 
    (func_name, error_desc, error_code)
 
where func_name is the name of the API function that
raised the error, error_code is one of the MSP430_EXC_*
constants, and error_desc is a string description of the
error.
 
Python callbacks may raise this exception in one of two
ways: with a single error_code argument, or with three
arguments. In the latter case, the third argument is
returned to the C API as the exception (the first two
arguments are ignored). If this protocol is not followed
or if python callbacks raise some other exception, the C
API gets a generic MSP430_EXC_PYTHON_FAULT.
 
The CPU object contains a 4-tuple describing the last
error:
 
    (error_code, detail_1, detail_2, line#)
 
where error_code is an MSP430_EXC_* code, detail_* are
word values describing the context (see the C source), and
line# is the line number in soft430.c where the error was
raised.

set_mem(...)
set_mem(func)
 
Set the memory handler to func. When memory is
accessed, func will be called as:
 
    val <- func(addr, val, isWrite, isWord)
 
and must return an integer or raise an exception.
addr is the mem address to be accessed. val is a
byte value to be stored if isWrite is nonzero.
isWrite is nonzero if a write is being requested;
it is zero for read requests. isWord is zero if a
byte access is requested and nonzero for a word access.
 
For word accesses, the high byte (at the odd address)
will be requested first, followed immediately by a
request for the low byte (at the even address). this
second request does not occur if an exception is
raised during the first request.
