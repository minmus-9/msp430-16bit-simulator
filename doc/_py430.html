This module defines a high-performance MSP430F1611
profiling object; it is intended to be used in conjunction
with the _soft430 module.

bind(...)
bind(msp430, ary) -> obj
 
Bind the MSP430 object as returned by _soft430.new
to the type-L array object ary. CPU execution will
update the profiling data in ary. The array map in
ary is as follows:
 
    start    length    contents
   ======    ======    ==================================
        0        16    register values
       16     49152    program rom
    49168     10240    ram
    59408     32768    pc counts by word address
    92176         3    sp info
    92179        28    counts by instruction
    92207        28    ticks by instruction
    92235     65536    address space accesses
   157771     32768    jumps to address
   190539     32768    calls to address
   223307     65536    breakpoints by address
   288843     32768    total ticks per pc address
 
the sp (r1) info consists of the following three slots:
 
   0 = nonzero if sp has been set
   1 = lowest sp value
   2 = highest sp value
 
NOTE: The CPU register, memory, and profiling handlers are
      replaced after the call to bind().
 
NOTE: If bind() fails, either none or all of these handlers
      will be reset to the default C handler function.
 
NOTE: When the returned object is destroyed, the CPU will
      be rebound to the default C handler functions.
